# 117 JS 1 DOM

## JavaScript

- 현재는 Chrome, Firefox, Safari, Microsoft Edge 등 다양한 웹 브라우저가 경쟁, 모바일 등 시장 다양화
- 과거에는 웹페이지의 동적인 기능을 구현하는 용도로만 사용됨
- 이후 Node.js의 출시로 인해 브라우저 외부에서도 실행 가능해져 서버 사이드 개발에도 사용되기 시작
- 다양한 프레임워크와 라이브러리들이 개발되면서, 웹개발 분야에선 필수적인 언어로

## 변수

### 작성 규칙

- 문자, $, _ 로 시작
- 대소문자 구분
- 예약어(for, if, function) 사용불가

**Name Convention**

- 카멜케이스 : 변수, 객체, 함수에 사용
- 파스칼케이스 : 클래스, 생성자에 사용
- 스네이크케이스 : 상수에 사용

## 변수 선언

### let

- 재할당 가능, 재선언 불가능
- 블록 스코프를 갖는 지역 변수를 선언

### const

- 재할당 불가능, 재선언 불가능
- 블록 스코프를 갖는 지역 변수를 선언

( 블록스코프 : 중괄호{} 내부를 가리킴, 블록스코프를 가지는 변수는 블록 바깥에서 접근 불가 )

### var

- 재할당, 재선언 가능
- 함수 스코프를 가짐
- 변수 선언시 키워드를 사용안하면 자동 var 가 됨
- 선언 위치와 관계없이 스코프 최상단에서 선언된 것처럼 도작
    
    (호이스팅 : 변수 선언문이 코드의 최상단으로 끌어올려지는 듯한 현상)
    

> const 를 기본으로 사용할 것
> 
- 코드의 의도 명확화
- 버그 예방
- 필요한 경우에만 let으로

DOM 트리

> DOM : 문서의 요소들을 객체로 제공하여 다른 프로그래밍 언어에서 접근하고 조작할 수 있는 방법을 제공하는 API
> 

Node : DOM의 기본 구성 단위

NodeList : DOM 메서드를 사용해 선택한 Node의 목록

Element : DOM 트리에서 HTML 요소를 나타내는 특별한 유형의 Node

Parsing : 브라우저가 문자열을 해석하여 DOM Tree로 만드는 과정

조작 순서

1. 요소를 선택! (또는 탐색!)
2. 선택된 요소의 콘텐츠 또는 속성을 조작
